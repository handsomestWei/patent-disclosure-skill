# 边界与输入录入（A）

A 阶段：用最少问题收敛案件边界。信息不全可跳过，由 Agent 后续推断并注明假设。

## 执行清单（逐项完成）

1. [ ] 出示 Q1：是否有参考文件？（设计文档、代码、PPT、现有交底书等；有→路径，无→"无"）
2. [ ] 出示 Q2：简要说明主要发明点和技术特征、解决的技术问题、达到的技术效果（从以下角度任选 1-3 描述核心思路即可）：
   （1）要解决什么技术问题？（2）核心的技术手段或创新点？（3）与现有方案的区别或改进？（4）达到了什么技术效果？
3. [ ] 出示 Q3：倾向的权利要求类型？（方法/系统/机构/光学/电路/综合/暂不确定；不确定→"暂不确定"，Agent 后续根据方案特征建议）
4. [ ] 记录每个回答来源（用户直接提供 / 文件推断）
5. [ ] 3-6 bullet 汇总已确认项，注明信息缺口
6. [ ] 按路由分支决定下一步：
   - Q1「无」→ 跳 B 直入 C，Q2/Q3 作为 C 阶段种子输入
   - Q1「有」→ 入 B 扫描 → 扫描结果含 Q2/Q3 信息则直入 C，否则补问

## 出口条件（声明"A 阶段完成"前必过）

- [ ] Q1-Q3 已回答或注明缺口
- [ ] 路由方向已确定
- [ ] 汇总 bullet 已在对话中呈现
