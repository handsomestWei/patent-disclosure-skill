# 边界与输入录入（A）

## 用途

在开始专利挖掘与交底书撰写前，用少量问题收敛案件边界。信息不全时可跳过，由 Agent 后续基于项目材料推断并**注明假设**。

## 问题序列

### Q1：参考文件

```
是否有参考文件（设计文档、代码、PPT、现有交底书等）？
（有就提供路径或文件名；没有就写「无」。）
```

### Q2：发明概述

```
请简要说明主要的发明点和技术特征、解决的技术问题、达到的技术效果。
（从以下角度任选 1-3 个来描述，不需要完整方案，几句话描述核心思路即可：
（1）要解决什么技术问题？
（2）核心的技术手段或创新点是什么？
（3）与现有方案相比有什么不同或改进？
（4）达到了什么技术效果？）
```

### Q3：专利类型倾向

```
是否有倾向的权利要求类型？
（方法 / 系统 / 机构 / 光学 / 电路 / 综合 / 暂不确定——不确定就写「暂不确定」，Agent 会根据方案特征后续建议。）
```

## 路由逻辑

```
Q1 回答「无」
  → 继续问 Q2、Q3
  → 跳过 B，直接进入 C（patent_points_analyzer.md）
  → Q2、Q3 的回答作为 C 阶段候选专利点的种子输入

Q1 回答「有」（提供文件路径）
  → 直接进入 B（project_scan.md）扫描文件
  → 扫描完成后判断：扫描结果是否已包含 Q2（发明概述）和 Q3（专利类型）的信息？
      • 已包含 → 直接进入 C，无需追问
      • 不完整 → 补充提问 Q2 和/或 Q3，用户回答后进入 C
```

## 汇总

将已确认项用 3–6 行 bullet 复述给用户，注明信息来源（用户直接提供 / 从文件中扫描提取），再按上述路由逻辑分支。
