# 项目文档扫描（B）

B 阶段：按优先级扫描并提取可专利化内容。仅当 A.Q1 有参考文件且非迭代纠错时执行。

## 扫描优先级

| 优先级 | 文档类型 | 关注内容 |
|--------|----------|----------|
| 1 | 专利相关文档 | 专利点分析、已有交底书、创新点汇总 |
| 2 | 详细设计/方案文档 | 详细设计、流程图、技术对比分析 |
| 3 | 核心实现代码 | 算法策略、业务逻辑、规则引擎、接口设计、状态机、调度机制等 |
| 4 | 系统设计文档 | 系统设计、架构说明、模块划分、数据流 |

## 执行清单（逐项完成）

1. [ ] Glob/列举项目目录，定位优先级 1-4 文档（大仓库先搜索关键词再精读）
2. [ ] `.docx`/`.pptx` → 先 `python3 tools/docx_to_md.py` / `tools/pptx_to_md.py` 转 Markdown 再读（含 `_media/` 嵌入图）
3. [ ] `Read` 转换后的 `.md` 及原有 `.md`/源码，同等对待
4. [ ] 记录引用路径（用于交底书脱敏表述）
5. [ ] 输出扫描摘要（技术方向·可专利点·信息缺口）
6. [ ] 判断扫描结果是否已覆盖 A.Q2（发明概述）和 A.Q3（专利类型）：已覆盖→直入 C；不完整→补问后入 C

## Office 文档转换

1. Glob `*.docx`、`*.pptx`（含子目录）
2. 执行转换（`${CLAUDE_SKILL_DIR}` 为技能根目录）：
   ```bash
   python3 ${CLAUDE_SKILL_DIR}/tools/docx_to_md.py -i "<路径>.docx" -o "<路径>.md"
   python3 ${CLAUDE_SKILL_DIR}/tools/pptx_to_md.py -i "<路径>.pptx" -o "<路径>.md"
   ```
3. `Read` 生成的 `.md`，与原有文件同等对待
4. 解析重点：表格、编号列表、PPT 每页标题与正文、备注

> `.doc`/`.ppt` 旧格式须先在 Office/WPS 中另存为 `.docx`/`.pptx`。

## 图片与裸图目录

- 独立 `.png`/`.jpg`/`.webp` 等不作为 B 必须逐个识图的对象（通常与 Office 内嵌图重复）
- 例外：用户点名某图，或某图未出现在任何 `_media` 中且对专利点明显关键

## 出口条件（声明"B 阶段完成"前必过）

- [ ] 优先级 1-4 文档均已扫描并阅读
- [ ] Office 文档已转换并阅读
- [ ] 扫描摘要已输出，A.Q2/Q3 覆盖情况已判断
