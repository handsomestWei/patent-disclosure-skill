# 专利点挖掘、查新与交互讨论（C）

C 阶段是整篇交底书的核心讨论环节。核心理念：**检索查重**是前置门禁，坚决避开已有专利点，基于差异化提出反问，必要时借鉴其他行业类似专利挖掘新的金点子。

**Agent 须保持独立判断**——与查新证据/技术事实冲突时给出具体证据和理由，不盲从用户。

输入来源（按优先级合并）：A 的 Q2/Q3 + B 扫描结果 + 用户的专利点补充。

## 执行清单（C 阶段循环；用户确认结论摘要后退出）

1. [ ] **前置询问**："是否有专利描述或专利点的补充？"有→作为种子综合提炼；无→从 A+B 材料挖掘
2. [ ] **初选专利点**（仅首轮）：从材料提炼 3-5 候选点，每点含：技术背景·创新点·与现有技术初步区别·可实施性·至少 1 个查重关键词
3. [ ] **用户确认候选点**：采纳/新增/修改/删除。**禁止未确认直接查新**
4. [ ] **按需查新**（`prior_art_search.md`，优先国知局→异常再降级 WebSearch）：
   - 新增专利点 → **必须检索**
   - 删除专利点 → 跳过
   - 修改表述 → 判断技术实质是否改变（手段/范围/维度变→检索；仅措辞→跳过）
   - 已有查新结果且方向不变 → 复用，不重复
5. [ ] **检索结果汇总**（对话中呈现）：
   - 相关提案清单（专利号+标题+摘要概括+与本案重叠特征）
   - 应避开的已有专利点
   - 差异化空间（2-5 方向）
6. [ ] **用户判断**：对每件相似专利——放弃该方向 / 避让该专利
7. [ ] **避让策略讨论**（需避让时 `Read patent_avoidance_methods.md`）：
   - 确认专利核心技术特征 → 识别重叠区
   - 从十类避让法中挑 2-4 类 → 每类提 1-2 具体规避方向
   - 必要时引入其他行业类似专利的解题逻辑作类比启发
   - **反向提问发明人**：每轮 2-5 具体问题，基于查新中发现的技术空白
   - 产生新方向→回到步骤 2（标记需检索）
8. [ ] **替代方案与保护点初议**：
   - 替代方案：2-4 方向，各有优缺点和适用条件，与主方案模块/流程/参数对应
   - 保护点：初定 2-5 核心点，可追溯到实施方式/公式/参数/流程/模块
9. [ ] **结论摘要确认**（见下方格式）→ 用户确认→退出循环，入 D

**循环控制**：新增→必须检索；删除→跳过；修改→判断实质；确认结论摘要→退出。讨论已充分但用户未确认时主动提醒。

## 避让分析输出格式

```markdown
### 专利 CNXXXXXXXXXA 的避让分析
**重叠技术特征**：[与本案重叠的特征]
**建议避让路径**：
- **[方法X]**：本案可通过……绕开。具体……。
- **[方法Y]**：本案可通过……绕开。具体……。
- **跨行业类比（如有）**：[其他行业类似解题逻辑]
**核心差异化路径**：[一句话总结]
```

## 结论摘要格式

```markdown
## 专利点挖掘结论摘要

### 确认的专利点
1. **[名称]**：……（创新点简述）

### 与现有技术的本质区别
- 区别于 CNXXXXXXXXXA（……）：本案通过……实现差异化。

### 应避开的已有方向
- ……（已被 CNXXXXXXXXXA 等覆盖）

### 最终保护点
（1）……（对应第 X 专利点）

### 可选替代方案
- 方案 A：……（优缺点/适用条件）
```

## 补充：多篇交底书

默认优先产出最有价值的一篇。若用户明确要求多篇：先列多篇大纲（标题+核心区别），再约定撰写顺序。

## 出口条件（声明"C 阶段完成，进入 D"前必过）

- [ ] 全部候选专利点已有查新结论（新增已检索，删除已标记）
- [ ] 每件需避让的专利已有具体规避分析（用避让法，非空泛）
- [ ] 替代方案与保护点已与用户讨论确认
- [ ] 结论摘要（专利点·区别·避让方向·保护点·替代方案）已输出并得到用户确认
- [ ] 用户明确表示"确认/可以/没问题/进入下一步/开始写交底书"
